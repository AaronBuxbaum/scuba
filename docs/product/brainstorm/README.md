# Brainstorm

Idea exploration, not commitments. These documents map the space of things Scuba *could* build so the
roadmap and [next-steps](../next-steps.md) can be chosen deliberately. Nothing here is scheduled until
it moves into the roadmap with a milestone and a note.

Each document takes a **different lens** on the same product so the same idea can be re-seen from more
than one angle. Read the lens that matches what you're trying to decide.

## How to read an idea here

Every idea is tagged so you can triage fast:

- **Effort** — S (days), M (a milestone slice), L (multi-milestone).
- **Pillar** — bookings / waivers / certs / gear / manifests / cross-cutting.
- **Bet size** — *quick win* (obvious value, low risk) vs *big bet* (could define the product, higher risk).

Ideas are graded against the north star ([next-steps](../next-steps.md)): does it reduce staff
coordination work, raise diver confidence, make departure safer, or speed agent delivery? An idea that
does none of those is noted as such and parked.
